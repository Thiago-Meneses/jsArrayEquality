Array Equality

Examples

Same Memory Address:

Both are assigned 5 as their value

However, their types are not the same

                                        
let a = [1, 2]
let b = a

console.log(`Double Equals: ${ a == b }`)
console.log(`Triple Equals: ${ a === b }`)

Console:

                                        
Double Equals: true
Triple Equals: true

Why are both logs true?

They are both true because in this example [1, 2] was assigned to a and, right after that, a was assigned to b

a was assigned with a Memory Address, this same memory address was the one assigned to b

Both variables have been assigned the same Memory Address, this is going to be their actual value, they are equal.

Different Memory Address

Both are assigned 5 as their value

However, their types are not the same

                                        
let a = [1, 2]	//0x01 - Memory Address Example
let b = [1, 2]	//0x02 - Memory Address Example
//0x01 is not the same as 0x02


console.log(`Double Equals: ${ a == b }`)
console.log(`Triple Equals: ${ a === b }`)

Console:

                                        
Double Equals: false
Triple Equals: false

So why are both logs false now?

This happened because even if both variables have 2 arrays that are equal in their values, each one of these arrays will occupy a different Memory Address

a and b were assigned to different addresses.

Even if both memory addresses have the same values, they are different addresses.

It's like having 2 identical cars in a parking lot, they can be the same model, but they are not in the same parking space

More Examples

Both are assigned 5 as their value

However, their types are not the same

                                        
let a = [1, 2]
console.log(`Value before: ${a}`)


function add( array, element ) { 	//( 0x01, 3 )
    array.push(element);
    let newArray = array;
    return newArray;
}

console.log(add( a, 3 ))
console.log(`Value after: ${a}`)

Console:

                                        
[1, 2]
[1, 2, 3]

The actual value being passed to the add function is the Memory Address that add holds.

If, for example, 0x01 is the Memory Address, this is what add receives as the first argument

When we call the add function, we end up also changing the a variable

After calling the function, a changed it's value from [1, 2] to [1, 2, 3]

We changed an array that was outside the function

This is bad and should be avoided.

                                        
let a = [1, 2]
console.log(`Value before: ${a}`)


function add( array, element ) { 	//( 0x01, 3 )
    array = [element] // array = 0x02
}

add( a, 3 )
console.log(`Value after: ${a}`)

Console:

                                        
[1, 2]
[1, 2]

When we use the equal sign, we are actually copying the value of the array from the right and transfering it to the array in the left

This is the catalyser that has made the Memory Address change from 0x01 to 0x02

And because of this 'array' no longer references the external 'a' variable, which we were originally modifying with the push command

Remember that everytime we use equal, we are reassigning a value, and this means we change memory addresses
